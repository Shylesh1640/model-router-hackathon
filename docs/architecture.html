Model Router — Architecture

Cost-optimized LLM routing · Launch Hook · Benchmark-Aware · Task Decomposition

Classification Layer

  • • Complexity: simple / medium / hard
  • • Task: code, reasoning, general
  • • Method: heuristic + optional embeddings
  • • Fallback: keyword/regex when no embedder

Routing Engine

  • • CostRouter maps complexity to tier
  • • Benchmark-aware: picks best model for task type
  • • Confidence override: <0.4 → bump tier
  • • Cascade: self-verify → escalate if unsure

Task Decomposer

  • • Splits multi-step queries into sub-tasks
  • • Each sub-task routed independently
  • • Cheap sub-tasks → fast model
  • • Hard sub-tasks → deep model
  • • Supports sequential, parallel, pipeline

Launch Hook + Benchmarks

  • • Scrapes OpenRouter on startup
  • • Resolves SWE-bench, MMLU, HumanEval scores
  • • Fills BenchmarkProfile per model
  • • Feeds into CostRouter as support metric
